# Docs website

Use the Docs website for formal TB information that should be easy to browse, search, and share.

## Good content for Docs

- TB operating procedures
- M&E indicator definitions and data-quality guidance
- Reporting templates and calendars
- Approved policies and decision records
- Meeting or review packs that must be retained

## Publishing standard

Every formal document should show:

- A clear title
- The document owner
- The approval or review date
- Its current status
- Links to related Issues or decisions when useful

## How updates work

Documentation maintainers edit the Markdown source in the repository. GitHub automatically rebuilds and publishes the site after approved changes reach the default branch.

!!! note
    Docs is the single source of truth for approved TB knowledge. Discussions, Issues and the Project board link back to it.
