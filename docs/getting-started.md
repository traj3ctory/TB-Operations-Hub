# Get started

Use the TB workspace through your web browser. You do not need Git, a terminal, or a code editor.

## Before you begin

- Sign in to GitHub.
- Open the [TB Operations Hub](https://github.com/traj3ctory/TB-Operations-Hub).
- Confirm that you can see **Issues**, **Discussions**, and **Projects**.
- Bookmark this documentation site.

## Choose the right place

| If you need to… | Go to… |
|---|---|
| Read or publish a formal TB document | Docs website |
| Request a change or report a problem | Issues |
| Ask a question or share an announcement | Discussions |
| Check delivery status | Projects |

!!! tip "Still unsure?"
    Start a Discussion. A coordinator can turn it into an Issue when a tracked action is required, then publish the approved outcome in Docs.

## Pilot contacts

The TB workspace should have one or two maintainers and at least one TB coordinator. Maintainers manage settings and access; coordinators review submissions and keep information current.
